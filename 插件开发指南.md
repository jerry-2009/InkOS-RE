# InkOS 插件开发指南

本文档说明如何为 InkOS 开发应用和系统插件。当前插件 API 以轻量、同步、单前台应用为核心，适合 ESP32-C3 + MicroPython 这类资源受限环境。

## 1. 插件目录结构

一个插件必须是一个独立目录，至少包含：

```text
my_plugin/
  manifest.json
  main.py
```

可选结构：

```text
my_plugin/
  manifest.json
  main.py
  icon.txt
  assets/
    data.txt
```

插件可放置在：

```text
/apps                 # 内置或普通应用
/plugins              # 系统插件
/sd/plugins           # SD 卡插件，适合用户扩展
```

## 2. manifest.json

`manifest.json` 用于声明插件元数据。示例：

```json
{
  "id": "example.clock",
  "name": "Clock",
  "version": "1.0.0",
  "entry": "main.py",
  "icon": "icon.txt",
  "api": 1,
  "author": "Your Name",
  "category": "app",
  "permissions": ["time", "settings"],
  "autostart": false,
  "min_inkos": "0.1.0"
}
```

字段说明：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `id` | 建议必填 | 全局唯一插件 ID。未设置时使用目录名。 |
| `name` | 建议必填 | Launcher 中显示的名称。未设置时使用 `id`。 |
| `version` | 建议必填 | 插件版本。建议使用语义化版本。 |
| `entry` | 否 | 入口文件，默认 `main.py`。只能是插件目录内相对路径。 |
| `icon` | 否 | 16x16 图标文件路径。无效或缺失时使用默认图标。 |
| `api` | 建议必填 | 目标插件 API 版本，当前为 `1`。 |
| `author` | 否 | 作者信息。 |
| `category` | 否 | `app` 或 `system`。默认 `app`。 |
| `permissions` | 否 | 插件声明需要的系统能力。当前主要用于说明和约定。 |
| `autostart` | 否 | 是否在系统启动时自动实例化并调用 `on_start()`。 |
| `min_inkos` | 否 | 最低 InkOS 版本。 |

安全限制：

- `entry` 不能包含 `..`。
- `entry` 不能以 `/` 开头。
- `icon` 不能包含 `..`，也不能以 `/` 开头。

## 3. 最小插件

`main.py` 必须提供 `create_app(context)`：

```python
from inkos.app.base import App
from inkos.events import ACTION_BACK


class HelloPlugin(App):
    app_id = "example.hello"
    name = "Hello"

    def on_event(self, event):
        if event.action == ACTION_BACK:
            self.context.exit_app()

    def render(self, display):
        display.line("Hello InkOS")
        display.line("")
        display.line("BACK to exit")


def create_app(context):
    return HelloPlugin(context)
```

对应 `manifest.json`：

```json
{
  "id": "example.hello",
  "name": "Hello",
  "version": "1.0.0",
  "entry": "main.py",
  "api": 1,
  "author": "Your Name",
  "permissions": [],
  "autostart": false,
  "min_inkos": "0.1.0"
}
```

## 4. 生命周期

插件类建议继承 `inkos.app.base.App`。可实现的方法如下：

```python
class App:
    def on_start(self):
        pass

    def on_resume(self):
        pass

    def on_pause(self):
        pass

    def on_stop(self):
        pass

    def on_event(self, event):
        pass

    def render(self, display):
        pass
```

生命周期含义：

| 方法 | 调用时机 | 用途 |
| --- | --- | --- |
| `on_start()` | 插件实例创建后首次启动 | 初始化状态、注册 HTTP 路由、读取资源 |
| `on_resume()` | 插件进入前台 | 恢复界面状态 |
| `on_pause()` | 插件离开前台 | 暂停前台交互、保存轻量状态 |
| `on_stop()` | 普通插件退出时 | 释放资源、关闭临时连接 |
| `on_event(event)` | 收到按键事件 | 处理上、下、确认、返回等动作 |
| `render(display)` | 系统刷新屏幕时 | 绘制当前界面 |

普通前台应用退出时，内核会调用 `on_pause()` 和 `on_stop()`，然后返回 Launcher。被记录在 `kernel.system_apps` 中的系统应用或自启动插件通常只暂停，不立即销毁。

## 5. AppContext

`context` 是插件访问系统能力的主要入口：

```python
context.kernel      # 内核对象
context.app_id      # 当前插件 ID
context.app_dir     # 当前插件目录
context.source      # flash / system / sd
context.display     # 当前显示驱动
context.logger      # 日志服务
context.settings    # 设置服务
context.http        # HTTP 文件服务
```

常用方法：

```python
context.request_render()
context.exit_app()
```

使用建议：

- 需要重绘界面时调用 `context.request_render()`。
- 需要退出当前插件时调用 `context.exit_app()`。
- 不建议插件直接调用 `kernel.render()`，除非确实需要系统级刷新。
- 插件不应长期保存硬件驱动的私有状态。

## 6. 输入事件

事件定义位于 `inkos/events.py`：

```python
ACTION_UP = "UP"
ACTION_SELECT = "SELECT"
ACTION_DOWN = "DOWN"
ACTION_BACK = "BACK"
ACTION_BACK_LONG = "BACK_LONG"
```

示例：

```python
from inkos.events import ACTION_BACK, ACTION_DOWN, ACTION_SELECT, ACTION_UP


def on_event(self, event):
    if event.action == ACTION_UP:
        self.index -= 1
        self.context.request_render()
    elif event.action == ACTION_DOWN:
        self.index += 1
        self.context.request_render()
    elif event.action == ACTION_SELECT:
        self.select_current()
    elif event.action == ACTION_BACK:
        self.context.exit_app()
```

`ACTION_BACK_LONG` 当前由内核优先处理，用于切换休眠或屏保状态，普通插件通常不会收到该事件。

## 7. 显示与绘制

`render(display)` 中通过显示驱动绘制界面。当前可用绘图接口包括：

```python
display.width
display.height
display.clear(color=1)
display.set_line_y(y)
display.text_width(value)
display.text(x, y, value, color=0)
display.line(value="")
display.hline(x, y, width, color=0)
display.vline(x, y, height, color=0)
display.rect(x, y, width, height, color=0, fill=1)
display.icon(x, y, bitmap, color=0)
display.bitmap_1bit(x, y, width, height, bitmap, color=0)
```

建议：

- 优先使用 `display.line()` 绘制简单列表和文本页面。
- 页面切换、弹窗关闭、首次进入应用时可请求全刷。
- 光标移动、状态变化等小更新交给系统默认刷新策略处理。
- 不要在插件中直接调用 `display.show()`，由内核统一提交刷新。
- 避免在 `render()` 中读取大文件、联网或执行耗时计算。

## 8. 设置读写

设置服务通过 `context.settings` 访问：

```python
value = self.context.settings.get("my_plugin.enabled", False)
self.context.settings.set("my_plugin.enabled", True)
self.context.settings.toggle("my_plugin.enabled")
```

注意：

- `set()` 和 `toggle()` 会尝试保存到 `/sd/data/settings.json`。
- 插件自定义设置建议使用插件 ID 前缀，例如 `example.clock.format`。
- 不要覆盖系统设置键，除非插件本身就是系统设置插件。

## 9. 日志

日志服务通过 `context.logger` 使用：

```python
self.context.logger.info("plugin started")
self.context.logger.error("operation failed")
self.context.logger.exception("callback failed", exc)
```

建议：

- 插件启动、异常、网络失败、文件写入失败应记录日志。
- 不要在高频刷新中持续写日志，避免影响性能和存储寿命。

## 10. HTTP 页面与路由

插件可在 `on_start()` 中注册 HTTP 路由：

```python
from inkos.services.http_request import Response


class ConfigPlugin(App):
    def on_start(self):
        self.context.http.route(
            "/app/example.config",
            self._http_config,
            methods=("GET", "POST"),
            title="Example Config",
        )

    def _http_config(self, request, conn=None):
        if request.method == "POST":
            self.context.settings.set("example.value", request.form("value", ""))
            return Response.redirect("/app/example.config")

        value = self._html_escape(self.context.settings.get("example.value", ""))
        body = "<main><h2>Example</h2><form method='post'><input name='value' value='%s'><button>Save</button></form></main>" % value
        return Response.html(self.context.http.page(body))

    def _html_escape(self, value):
        return str(value).replace("&", "&amp;").replace('"', "&quot;").replace("<", "&lt;").replace(">", "&gt;")
```

HTTP 能力依赖文件服务启动。插件注册的 `/app/...` 路由会显示在文件服务首页的 Apps 区域。

安全建议：

- 所有用户输入都应做 HTML 转义。
- 避免在 HTTP 回调中执行长时间阻塞操作。
- 写入设置后使用重定向，避免刷新页面重复提交。
- 文件上传和删除能力只应在可信网络中使用。

## 11. 自启动插件

设置：

```json
{
  "autostart": true
}
```

自启动插件会在内核启动期间被实例化并调用 `on_start()`。适合：

- 注册 HTTP 配置页面。
- 准备后台服务状态。
- 初始化低频系统服务。

不适合：

- 启动后立即联网阻塞。
- 创建死循环。
- 高频轮询。
- 在 `on_start()` 中直接刷新屏幕。

## 12. 系统插件与普通应用

`category` 默认是 `app`。设置为 `system` 后，Launcher 会将其放入系统分类：

```json
{
  "category": "system"
}
```

建议分类：

- 普通用户功能：`app`
- 设置、文件服务、屏保、对时、电源管理：`system`

## 13. 权限声明

当前权限声明主要用于说明插件能力，并不构成强隔离沙箱。可用约定包括：

```text
display
input
settings
http
network
time
fs.read
fs.write
screensaver
power
system
```

建议按实际使用声明权限。例如联网对时插件：

```json
{
  "permissions": ["network", "time", "http"]
}
```

## 14. 图标

插件可提供 16x16 单色图标，并在 `manifest.json` 中声明：

```json
{
  "icon": "icon.txt"
}
```

图标加载由 `inkos.ui.icons.load_icon()` 处理。若图标缺失、路径不安全或解析失败，系统会使用默认应用图标。

## 15. 性能与内存约束

ESP32-C3 + MicroPython 的 RAM 有限，插件应遵守以下原则：

- `import` 阶段只定义类和函数，不执行耗时逻辑。
- 不在 `on_event()` 中做长时间阻塞操作。
- 不在 `render()` 中读大文件或联网。
- 大文件读取应分块处理。
- 页面列表只渲染可见项。
- 尽量复用字符串和小对象。
- 退出前释放临时引用，必要时让系统执行 `gc.collect()`。

## 16. 错误处理

插件异常会被内核捕获并尝试返回 Launcher。插件仍应主动处理可预期错误：

```python
try:
    self.load_data()
except Exception as exc:
    self.context.logger.exception("load data failed", exc)
    self.message = "Load failed"
    self.context.request_render()
```

推荐策略：

- 用户可恢复的问题显示在界面上。
- 开发者需要排查的问题写日志。
- 网络和文件操作都应捕获异常。
- 不要让异常从 HTTP 回调中无控制地冒泡。

## 17. 文件访问建议

当前版本尚未提供专门的插件私有文件系统 API。建议插件只访问：

- 自身目录下的只读资源。
- `/sd/data/` 下以插件 ID 命名的数据文件或目录。

示例：

```python
def data_path(self, name):
    return "/sd/data/%s_%s" % (self.context.app_id.replace(".", "_"), name)
```

注意：

- 不要删除或覆盖系统目录。
- 不要使用用户输入直接拼接路径。
- 路径中应拒绝 `..`。

## 18. 开发检查清单

发布插件前建议检查：

- `manifest.json` 是合法 JSON。
- `id` 全局唯一。
- `entry` 文件存在。
- `main.py` 使用 UTF-8 编码。
- `create_app(context)` 存在并返回 App 实例。
- `ACTION_BACK` 能正常退出前台插件。
- `render(display)` 不调用 `display.show()`。
- 插件异常不会导致系统卡死。
- 如果注册 HTTP 路由，输入已做 HTML 转义。
- 如果写入设置，键名使用插件 ID 前缀。
- 如果使用 SD 卡文件，路径已做安全检查。

## 19. 完整示例：计数器插件

目录：

```text
counter/
  manifest.json
  main.py
```

`manifest.json`：

```json
{
  "id": "example.counter",
  "name": "Counter",
  "version": "1.0.0",
  "entry": "main.py",
  "api": 1,
  "author": "Your Name",
  "category": "app",
  "permissions": ["settings"],
  "autostart": false,
  "min_inkos": "0.1.0"
}
```

`main.py`：

```python
from inkos.app.base import App
from inkos.events import ACTION_BACK, ACTION_DOWN, ACTION_SELECT, ACTION_UP


class CounterApp(App):
    app_id = "example.counter"
    name = "Counter"

    def on_start(self):
        self.value = self.context.settings.get("example.counter.value", 0)

    def on_event(self, event):
        if event.action == ACTION_UP:
            self.value += 1
            self._save()
        elif event.action == ACTION_DOWN:
            self.value -= 1
            self._save()
        elif event.action == ACTION_SELECT:
            self.value = 0
            self._save()
        elif event.action == ACTION_BACK:
            self.context.exit_app()

    def render(self, display):
        display.line("Counter")
        display.line("")
        display.line("Value: %s" % self.value)
        display.line("")
        display.line("UP/DOWN change")
        display.line("SELECT reset")

    def _save(self):
        self.context.settings.set("example.counter.value", self.value)
        self.context.request_render()


def create_app(context):
    return CounterApp(context)
```

## 20. 兼容性说明

当前 InkOS 版本为 `0.1.0`，插件 API 版本为 `1`。公开发布插件时建议在 README 中标注：

```text
Requires InkOS >= 0.1.0
Plugin API: 1
```

未来若插件 API 出现不兼容变化，应提升 `api` 版本，并在插件管理器中做兼容性检查。
